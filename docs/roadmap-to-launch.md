# FlashNote AI リリースまでのロードマップ

**完成（v1.0 リリース）までに必要な残りの実装項目**

---

## 1. 必須機能 (Must Have)

これらはサービスのコア価値を提供するために不可欠です。

### 1.1 Stripe 決済機能の導入
- **目的**: マネタイズと利用制限。
- **実装内容**:
    - Stripe Checkout / Customer Portal の統合。
    - Core API に Webhook エンドポイントを追加し、課金ステータスをDBに同期。
    - Chrome拡張機能側で、無料/有料ユーザーの判別ロジックを追加。

### 1.2 エラーハンドリングと堅牢性 (Robustness)
- **目的**: ユーザーに「失敗」を見せない、あるいは適切に伝える。
- **実装内容**:
    - **QStash Dead Letter Queue (DLQ)**: 失敗したジョブの再試行設定。
    - **エラー通知**: 生成失敗時に、Chrome拡張機能側で「失敗しました」と表示し、再試行ボタンを出す。

---

## 2. 品質向上 (Quality Assurance)

### 2.1 E2E テスト (End-to-End)
- **目的**: 全体フローの動作保証。
- **内容**:
    1.  Chrome拡張機能でログイン。
    2.  YouTube動画で「生成」ボタン押下。
    3.  Core -> QStash -> Worker -> Groq -> Notion のフローが完走することを確認。
    4.  Notionにページが作成されていることを目視確認。

### 2.2 ユーザビリティテスト
- **目的**: 使い勝手の改善。
- **内容**:
    - 初見のユーザーが迷わずインストール〜利用開始できるか？
    - ローディング時間が長く感じないか？（UIポーリングの体感速度確認）

---

## 3. マーケティング & デプロイ (Launch)

### 3.1 本番環境へのデプロイ
- **Core / Worker**: Railway, Render, または Vercel (Python) へのデプロイ。
- **Database**: Supabase (本番プロジェクト) の設定。

### 3.2 ランディングページ (LP) 作成
- **目的**: ユーザー獲得。
- **内容**:
    - ヒーローセクション（キャッチコピー）。
    - デモ動画（YouTube埋め込み）。
    - 価格表 (Pricing)。
    - Chrome Web Store へのリンク。

### 3.3 Chrome Web Store への申請
- **内容**:
    - `manifest.json` の最終確認。
    - スクリーンショット、プロモーション画像の作成。
    - プライバシーポリシーの用意。

---

## 4. 優先順位の提案

まずは **「1.2 エラーハンドリング」** と **「3.1 本番デプロイ」** を優先し、**「自分自身が使える状態 (Dogfooding)」** にすることをお勧めします。

決済機能 (Stripe) は、ユーザーが増えてからでも遅くありません（最初は完全無料ベータ版として公開する戦略もあります）。
