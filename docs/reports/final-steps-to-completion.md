# TubeWiki 完成に向けた残りのステップ

## 現状の達成状況

主要な機能実装はほぼ完了しています。

- [x] **拡張機能の基盤**: Vite, React, TS, Manifest V3
- [x] **UI実装**: Popup, Content Script (Overlay + Shadow DOM)
- [x] **認証基盤**: Supabase Auth + Chrome Storage Adapter (Popup/Content/Background間でのセッション同期)
- [x] **バックエンド連携**: ノート作成・取得APIの実装と連携

---

## 残っているタスク (To-Do)

### 1. 動作検証 (End-to-End Testing)
コード上は連携されていますが、実際のブラウザ環境での通しテストが必要です。

- **認証フロー**: Popupでログイン -> Content Script (YouTube) で即座に認識されるか？
- **データフロー**: YouTubeで「Generate Wiki」 -> バックエンドでジョブ作成 -> ポーリング -> 完了表示。
- **エラーハンドリング**: ネットワークエラー、未認証時の挙動、APIエラー時のUI表示。

### 2. UI/UX のブラッシュアップ
機能は動きますが、ユーザー体験を向上させるための調整が必要です。

- **ローディング表示**: ノート生成中のプログレス表示（現在はスピナーのみ）。
- **Markdown表示**: ノート内容（Markdown）のレンダリングスタイル調整（`prose` クラスの適用確認）。
- **開閉アニメーション**: サイドバーの出し入れをスムーズに。

### 3. パッケージングとデプロイ準備
ストア公開や配布のための準備です。

- **ビルド**: `npm run build` で生成された `dist` フォルダの確認。
- **アイコン**: デフォルトのReactアイコンのままなので、TubeWiki用のアイコンに差し替え。
- **不要ファイルの削除**: `console.log` の削除など。

---

## 推奨される次のアクション

1.  **手動テストの実施**:
    -   Chromeの「拡張機能を管理」->「パッケージ化されていない拡張機能を読み込む」で `dist` を読み込み。
    -   YouTubeを開き、動作を確認。

2.  **バグ修正**:
    -   テストで見つかった課題（例: セッション同期のラグなど）を修正。

3.  **アイコン設定**:
    -   `public/icons` にアイコン画像 (16, 48, 128px) を配置し、`manifest.json` を更新。

これが完了すれば、v1.0 としてリリース可能な状態になります。
