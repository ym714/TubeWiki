---
description: 調査してドキュメントを作成する
---

ユーザーが調べて欲しいと思った内容を調査し，適切なタイトルで`docs`ディレクトリに`.md`形式で知見をドキュメント形式で保存してください．

ユーザーが調査を指示した場合，以下の手順で実行してください：

1. **調査内容の理解**: ユーザーが調べて欲しい内容を明確に把握する
2. **情報収集**: `web_search`ツールや`codebase_search`ツールを使用して関連情報を収集する
3. **ドキュメント作成**: 
   - `docs`ディレクトリが存在しない場合は作成する
   - 調査内容に基づいて適切なタイトルを決定する（kebab-case推奨）
   - `.md`形式でドキュメントを作成し，収集した知見を構造化して記述する
   - 必要に応じてコード例，参考リンク，注意事項などを含める
4. **ファイル保存**: `docs/<適切なタイトル>.md`として保存する
5. **Marp形式での記述**: ドキュメントはMarp形式（スライド形式）で記述する
   - フロントマターに `marp: true` を含める
   - 適切なヘッダー、フッター、テーマを設定する
   - ページネーションを有効にする
